CSS animations

    @keyframes.
    in this property we declare a variable at anywhere outside of the bracket and the there are two brackets inside it.
    1)from { }
    2)to { }
    in this we define from where the animation will start and where it will end.
    and then the variable in which that animation is stored that is called in the animation style (aniamtion: variable;).
    example:: 
                    .div{
                    animation: YaseenKaAnimation;
                }
                @keyframes YaseenKaAnimation{ 
                from{

                }
                to{
                    transform: translatex(1000px);
                }
            }
    Now the animation will not get applied to apply the animation we have to add the animation properties in the div in which we wish to apply the animations.
  1. animation.
    2. this is the tag in the CSS which is used to call the variable in which the animations are declared.
    it's example is given above in the (.div).
  2. animation-duration
    3. this is the time till which the animation will be executed. and it is represented in (s) seconds.
    syntax:: animation-duration: 2s;
  3. animation-iteration-count.
    4. this property is used to define how many times that particular animation will be held i.e it may be 1 time 2 times 3 times and the infinite times.
    syntax::
    animation-iteration-count: 1;
    animation-iteration-count: 2;
    animation-iteration-count: 3;
    animation-iteration-count: infinite;
  4. animation-timing-function.
    5. this function we have seen in the transition function i.e this has the values ease, ease-in, ease-out, ease-in-out, and the cubic-bazier property (which can be changed from the browser.)
    syntax:: transition-timing-function: ease;
  5. animation-delay.
    6. this property also we have seen in the transition topic this specifies the delay in the beginning of the animation after we reload the page.
    syntax:: animation-delay: 1s;
  6. animation-direction.
    7. this specifies the direction of the animation. It has several values :: reverse, alternate-reverse etc.
    syntax::
    animation-direction: reverse;
    animation-direction: alternate-reverse;
  7. animation-play-state.
    8. this property makes the animation paused at the place where it is at the time of applying the property. and then after that animation will continue from that place after applying the property running.
    this consists two properties i.e running and pused.
  8. animation short-hand.
    9. the shorthand of the animation property is the application of all the animation properties in one property i.e animation: ;
    in the vs code the vs code assist us to fill the all properties of animation in one tag.
    i.e animation: name duration timing-function delay iteration-count direction fill-mode;
    syntax ::
    animation: (name) (duration) (timing-function) (delay) (iteration-count) (direction) (fill-mode);
    example ::
    animation: YaseenKaAnimation 2s ease-in-out 1s infinite reverse;
    we can remove the property as we want for example we can remove the delay and fill-mode.
  9. animation-fill-mode.
    10. by default this property have the value none if we don't apply this property to the element.
    it has the values::
    • animation-fill-mode: none;
      • this property is by default applied on the animations.
    • animation-fill-mode: forwards;
      • in this animation property the object on which the animation is applied that will retain the final keyframe value at the end of the animation it will not go back to its initial position after the animation ends.
    • animation-fill-mode: backwards;
      • in this animation property the object to which the animation is applied that will retain the initial keyframe value after the animation ends.
      for more refer the video from @13:15.
      animation: Yaseen2 2s ease-in-out 1s 2 forwards;

      animation: Yaseen2 2s ease-in-out 1s 2 backwards;
  10. defining the states of the animations.
    11. apart from adding the (from and to) in the @keyframes we can add that what will happen after the animation is percentage wise done.
    syntax:: 
    @keyframes Yaseen3 {
                10%{
                    transform: rotate(20deg);
                    transform: translatex(50%) translatey(50%);
                }
                30%{
                    transform: rotate(1turn);
                    transform: translatex(50%) translatey(50%);
                }
                70%{
                    background-color: blueviolet;
                    transform: translatex(100%) translatey(100%);
                }
                100%{
                    transform: scale(1.2);
                    transform: rotate(160deg);
                }
            }
     animation: Yaseen3 2s ease-in-out 1s infinite alternate-reverse;
  11. multiple animations.
    12. we can add the multiple animations in the single element to make it more attractive.
    we can add the multiple animations by coma (,) separated values.
    syntax::
    animation: YaseenKaAnimation 2s ease-in-out 1s infinite reverse, Yaseen2 2s ease-in-out 1s infinite reverse;
